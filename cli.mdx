## Overview

`fare-terminal` automates deploying a custom casino backend to production with zero manual infrastructure setup.

## Installation

```bash
npm install -g fare-terminal
```

Development install:

```bash
npm install
npm run build
npm link
```

## Commands

Binary: `fare`

### `fare init <casino-name>`

Initialize and deploy a new custom casino.

What it does:
- Clone backend template
- Interactive configuration
- Build TypeScript project
- Create Fly.io app and provision Redis, object storage, Postgres
- Set env secrets, run migrations, deploy

Example:

```bash
fare init my-awesome-casino
```

Prompts include Solana owner wallet, RPC provider, and JWT secret.

### `fare deploy`

Rebuild and redeploy existing casino.

```bash
cd ~/.fare-casinos/my-awesome-casino
fare deploy
```

### `fare status`

Check deployment health (api, ws, processor) and service URLs.

```bash
cd ~/.fare-casinos/my-awesome-casino
fare status
```

### `fare logs [service]`

Stream logs from Fly.io services (`api`, `ws`, `processor`).

```bash
fare logs        # all services
fare logs api
fare logs ws
fare logs processor
```

### `fare cleanup`

Delete local casinos and Fly.io resources. Destructive — for development only.

```bash
fare cleanup
fare cleanup --org <org>
```

## Prerequisites

- Node.js >= 18
- Git
- Fly.io CLI (`fly` or `flyctl`) installed and authenticated

## Fly.io Authentication

```bash
fly auth login
fly orgs list
```

During `fare init`, you’ll be prompted to select an organization if multiple are available.

## Casino Directory Layout

Casinos are stored under `~/.fare-casinos/<casino-name>`:

```text
~/.fare-casinos/
└── my-awesome-casino/
    ├── fare.config.json   # metadata (name, createdAt, path)
    ├── .env               # environment variables generated from the wizard
    ├── apps/              # backend monorepo (from template)
    └── ...
```

## Environment Variables

Generated `.env` includes:

- `CASINO_NAME`, `FRONTEND_URL`
- `SOLANA_RPC_URL`, `SOLANA_OWNER_ADDRESS`
- `HEARTBEAT_PRIVATE_KEY`, `JWT_SECRET`
- `DATABASE_URL`, `REDIS_URL` (set after provisioning)
- `NODE_ENV`, `PORT`, `HOST`
